# Epistemic Lens — Adversarial Red-Team Stress Test

> **Date:** 2026-05-08 · **Branch:** `claude/review-stress-test-06asX` · **Pinned at review:** `meta_version 2.0.1`
>
> Stance: adversarial red-team, requested by the author. Diplomacy stripped on purpose.
> Scope: idea/problem fit · methodology rigor · data/coverage integrity · results validity. Written critique only — no experiments executed.

---

## 1. Executive verdict

Epistemic Lens is **excellent infrastructure wrapped around a measurement instrument that does not measure what it claims**. The pipeline is well-engineered, the methodology pin is a real contribution, the citation-grounding validator blocks the worst LLM hallucinations, and the daily $0/mo cron is impressive.

But the central quantitative claims are confounded by three flaws that are visible in the code on the day of this review:

1. The pinned tokenizer regex `[A-Za-z]{4,}` (`meta_version.json:13`) produces **zero tokens for any non-Latin script**. Half of the 54 buckets — Russian, Chinese, Japanese, Korean, Arabic, Hindi, Hebrew, Greek, Thai — silently contribute empty vocabularies to every Jaccard, isolation, and exclusive-vocab number the system publishes.
2. The methodology pin hashes inputs but cannot pin Sonnet 4.6's weights. Anthropic ships model updates without weight-level versioning. Every longitudinal claim is confounded with model drift the pin is powerless against.
3. The framing prompt (`.claude/prompts/daily_analysis.md:76`) explicitly mandates: *"Frames are story-specific. Re-derive every time. Do NOT reuse labels across stories or across days."* This defeats longitudinal frame tracking — the exact use case the methodology pin is supposed to enable.

The author is also already aware of (1): the prompt's own example output annotates `"isolation is linguistic, not editorial"` (`daily_analysis.md:110`). The confound is acknowledged in the prompt and shipped anyway.

**Bottom line.** Two honest paths: (a) re-scope as a "curated, citation-grounded multilingual news reader" — modest, useful, defensible; or (b) keep the measurement claims but replace the cross-lingual metrics, adopt a validated frame codebook, add multi-LLM/multi-rater protocols, and stop calling longitudinal what is currently n≈5 over two months.

## 2. The four fatal flaws

### 2.1 The tokenizer literally cannot see non-Latin scripts

`meta_version.json:13` pins `"regex": "[A-Za-z]{4,}"`. The character class `[A-Za-z]` excludes:

| Script | Languages affected (in EL's bucket list) | Tokens produced |
|---|---|---|
| Cyrillic | Russian, Ukrainian, Bulgarian, Serbian | **0** |
| Han | Chinese (simplified + traditional), Japanese kanji | **0** |
| Kana | Japanese hiragana/katakana | **0** |
| Hangul | Korean | **0** |
| Arabic | Arabic, Persian/Farsi, Urdu | **0** |
| Devanagari | Hindi, Marathi | **0** |
| Hebrew | Hebrew | **0** |
| Greek | Greek | **0** |
| Thai | Thai | **0** |

It also drops Latin words containing diacritics: `café` is segmented as `caf` (3 chars, below `min_token_length=4`, dropped), `México` → `xico`, `Köln` → `ln` (dropped), `naïve` → `na` and `ve` (dropped). In `analytical/build_metrics.py:33-47`, `bucket_vocabularies()` concatenates `title + signal_text` per bucket and runs them through `meta.tokenize`. Non-Latin buckets therefore ship empty Counters.

**Consequence.** Every published `pairwise_jaccard`, `isolation`, `bucket_exclusive_vocab`, and `bucket_token_counts` row for these buckets is structurally meaningless. Russia–USA Jaccard is 0 by construction, not because they disagree. "Russia is the most isolated bucket" reduces to "Russian doesn't use Latin characters." The methodology pin guarantees this is **deterministically wrong** every day.

This is the worst kind of bug for a measurement project: silent, deterministic, framed as a feature (`stopword-filtered, plural normalization`), and reproducible to the byte.

### 2.2 The stopword file is English-only across 16+ languages

`stopwords.txt` (37 lines, 121 words) is entirely English: *the, and, that, with, from, this, …*. There is no list for `dello / della / sono / non` (Italian), `ist / und / der / die / das` (German), `está / pero / como` (Spanish), `est / dans / pour` (French), `de / em / não` (Portuguese), or any non-Latin language.

For the buckets that *do* produce tokens, this means common Italian/French/Spanish/German function words trivially clear the `bucket_exclusive_vocab` filter (`min_count=3`, `max_doc_freq=1`). The prompt's example finding (`daily_analysis.md:113-114`) — *"italy: ['guerra', 'accordo', 'uniti'] → war framing; Italian press treats this as conflict-not-deal"* — is generated by exactly this artifact:

- *guerra* = the Italian word for "war." Every Italian outlet uses it for any war story. It is exclusive to the Italy bucket because Italy is the only Italian-language bucket, not because Italian press uniquely chose conflict framing.
- *uniti* is the second word of *Stati Uniti* ("United States"). It is the country name, not a frame.
- *accordo* = "agreement / deal." Same translation artifact.

The system is publishing **language detection as framing analysis**, and the author has even written interpretive language ("war framing; conflict-not-deal") on top of it.

### 2.3 Methodology pin protects inputs but not the LLM weights

`meta_version.json:64-65` pins `"model": "claude-sonnet-4-6"` and `"prompts_hash"`. There is no API for pinning the *weights* of a hosted Anthropic model — model identifiers are aliases that move forward as Anthropic ships safety/quality updates. Sonnet 4.6 in May 2026 ≠ Sonnet 4.6 in November 2026 with respect to: instruction following, refusal behaviour, frame extraction style, quote selection bias, JSON formatting consistency.

Every longitudinal claim Epistemic Lens makes — "Hormuz had 14 buckets day 1, 6 by day 7" (README), "Italy's framing of Iran shifted from X to Y" — is **confounded with silent model drift** the pin cannot detect. The pin produces *false confidence*: a user reading `meta_version: 2.0.1` on two artifacts assumes they are comparable; they are not.

To fix this honestly the pin would need to: (a) sample the model with a fixed canary prompt set on every run and log the response distribution, (b) flag drift when responses diverge beyond some bound, or (c) switch to an open-weight model frozen by checksum (Llama, Qwen, Mistral). None of these are in place.

### 2.4 The prompt forbids the longitudinal frame tracking the pin promises

`daily_analysis.md:76` contains the rule:

> *"Frames are story-specific. Re-derive every time. Do NOT reuse labels across stories or across days. (Once we accumulate a few weeks of data we may pin a taxonomy; for now, free-form per story.)"*

This is internally contradictory with the methodology pin's stated purpose (`docs/METHODOLOGY.md`): enabling longitudinal claims like "this bucket's framing shifted from frame A on day 1 to frame B on day 7." If the label vocabulary is re-derived from scratch every day, *frame A on day 1 does not denote the same construct as frame A on day 7*, even if the strings match. The pin guarantees byte-stable inputs, then the prompt explicitly invites label drift on top.

Without a closed taxonomy (Entman 1993; Boydstun et al.'s 15-frame Policy Frames Codebook; Card et al.'s Media Frames Corpus), "frame" is whatever the LLM emits today. There is no codebook, no inter-coder reliability, no Krippendorff α. This is **LLM-generated literary criticism wearing the form of measurement**.

## 3. Idea / problem fit

### Comparable landscape

| System | What it does | Where EL falls short |
|---|---|---|
| **GDELT 2.0** | 15-min latency, 100+ langs, billions of events with CAMEO/GKG themes, public BigQuery | EL has 235 feeds and ~5 stories. GDELT laps it on every quantitative dimension. |
| **Mediacloud** | 1.7B+ stories, full text, network/topic analysis, since 2008 | Mediacloud already has the longitudinal corpus EL is trying to build by hand. |
| **Ground News / AllSides** | Bias rating, blindspot reports, consumer UX | Clearer mental model for non-academic users. EL's "frames/paradoxes/silences" needs a humanities seminar to interpret. |
| **V-Dem / Kontextlab** | Validated coding schemes, published inter-rater reliability | EL has neither codebook nor agreement metrics. |

### The user question

Nowhere in the repo is there a stated user, success metric, or theory of impact. "Source-transparent" is a producer's virtue. *What decision does anybody make differently because of an Epistemic Lens output?* If the answer is "feel sophisticated about media bias," that is entertainment, not infrastructure, and competes with Ground News, which has UX, scale, and a clearer offer.

The "$0/mo cost" framing is a feature **only if the output has non-zero validated value**. Cost-of-zero × value-of-zero is still zero.

## 4. Coverage / data integrity failures

### 4.1 The geographic-equality fallacy

The 54-bucket scheme treats one Italy (60M people, 4 outlets) as equivalent to one "Africa" bucket (1.4B people across 50+ countries, 6 feeds — admitted in `docs/COVERAGE.md`). Any aggregate "global framing" claim is dominated by Western European overrepresentation. There is no population weighting, no audience weighting, no per-bucket confidence interval that reflects the bucket's actual representativeness.

### 4.2 Extraction failure is non-random

`archive/extraction_test_2026-05-06.md` reports **74% FULL, 14% PARTIAL, 12% ERROR** (n=43, one per bucket). This is not noise:

- The outlets that *fail* extraction are the ones with paywalls, JS-rendered bodies, and aggressive anti-bot defenses: NYT, WaPo, WSJ, FT, Le Monde, Bild.
- The outlets that *succeed* are the open, easy-to-scrape long tail.
- Therefore the corpus is systematically biased **toward the long tail of journalism**, not the consensus mainstream — the opposite of what most consumers would assume "global media framing" means.

This is not a footnote; it inverts the population the system claims to describe.

### 4.3 Stub-only aggregator buckets corrupt vocab metrics

The 2026-05-06 baseline records: Reuters via Google News **41/50 stubs**, China Global Times **50/50 stubs**, plus several BBC feeds. `build_metrics.py:45` concatenates `title + signal_text`. For stub feeds, `signal_text` is the title or summary. **Title statistics are radically different from body statistics**: shorter, more SEO-optimized, with capitalized headline conventions, and trivially excluded by `[A-Za-z]{4,}` for any non-English title casing.

Any pairwise Jaccard between a stub bucket and a body bucket is comparing different linguistic objects. The methodology pin makes this byte-stable, not valid. These buckets should be **disqualified from quantitative comparison** until full bodies are obtainable, not merely footnoted.

### 4.4 RSS itself is a biased sample

RSS feeds are algorithmically curated by outlets. They typically: exclude opinion, exclude paywalled content, over-represent SEO-optimized stories, cap item counts (`max_items_per_feed=50` in the pin), and miss breaking-news inserts that bypass the CMS. *"What 235 RSS feeds emit"* ≠ *"what those outlets publish"* ≠ *"what their audiences read."* The selection function is opaque even to the project itself.

## 5. Results validity gaps

- **Citation grounding ≠ frame validity.** The Phase-4 validator (`analytical/validate_analysis.py`) checks that quotes exist at their claimed indices and that numbers match `metrics.json`. It does not check that the frame *label* is the right interpretation of the cited quotes. A correctly-cited quote can support a tendentious frame.
- **No inter-LLM agreement.** Run the same briefing through Sonnet 4.6, GPT-5, Gemini 3, and Llama 4. Report Krippendorff α on frame labels. Without this, every published frame is a **sample of size one**.
- **No test-retest stability.** Run the same briefing through Sonnet three times on the same day. Frame agreement should be high. The experiment is trivially cheap and not done. Its absence is damning for any project claiming measurement.
- **Single-rater editorial review.** PR review by the author is below content-analytic standards for any framing-analysis publication.
- **n ≈ 5 canonical stories over two months ≠ longitudinal dataset.** It is a case study. The "longitudinal" framing is overclaimed by at least an order of magnitude.
- **Internal inconsistency.** `meta_version.json:64` pins model `claude-sonnet-4-6`. The example output skeleton in `daily_analysis.md:126` shows `"model": "claude-haiku-4-5-20251001"`. The pin and its own usage prompt disagree.
- **Jaccard on token *sets* (`build_metrics.py:51`)**, not frequencies, makes the metric over-sensitive to rare long-tail tokens and equivalent for "USA mentions Iran twice" vs "USA mentions Iran 200 times."
- **DBSCAN `eps=0.35, min_samples=3` is unjustified.** No k-distance plot, silhouette analysis, or sensitivity sweep is documented. With 3–5 stories/day, the clustering is doing almost no work — it is largely aliasing the 5 hardcoded patterns in `canonical_stories.json`.

## 6. Five killer experiments (falsification roadmap)

These are concrete, cheap, and decisive. Each one can either kill or vindicate a specific claim.

| # | Experiment | What it tests | What "fail" looks like |
|---|---|---|---|
| 1 | Same briefing × Sonnet 4.6 × 3 runs same day, 1 run next day | Test-retest stability of frames | Frame-label Jaccard < 0.7 → longitudinal claims are noise |
| 2 | Translate every `signal_text` to English via NLLB-200, then re-run `build_metrics.py` | Whether "exclusive vocab" findings survive language-stripping | "guerra unique to Italy" disappears → metric was language ID |
| 3 | Same briefing × {Sonnet, GPT-5, Gemini 3, Llama 4} | Inter-LLM agreement on frame labels | Krippendorff α < 0.6 → "frame" is a model artifact |
| 4 | Pull GDELT GKG themes for the same 5 stories on the same dates | Marginal signal vs an established baseline | EL surfaces nothing GDELT misses → no novel contribution |
| 5 | Replay all 5 canonical stories in 30 days against current Sonnet endpoint, compare to original outputs | Detect silent model drift | Output divergence > threshold → pin is theatre |

Experiment 2 is the most important. Run it before defending any "exclusive vocabulary reveals X framing" claim publicly.

## 7. Alternatives matrix

| Component | Current | Alt A | Alt B | Alt C |
|---|---|---|---|---|
| Tokenizer | `[A-Za-z]{4,}` (drops all non-Latin) | **Unicode word boundaries + per-script regex** (pytroth, regex `\w` with UNICODE) | **Language-specific tokenizer per bucket** (jieba/sudachi/mecab/farasa) | **Subword tokenizer (XLM-R sentencepiece)** — lossless, language-agnostic |
| Stopwords | English-only, 121 words | **NLTK + spaCy stopword unions per language** | **Frequency-based per-corpus stopword filter** (top-N per bucket) | **Skip stopwords; use TF-IDF / log-odds** instead of raw counts |
| Cross-bucket metric | Jaccard on raw tokens | **Translate-to-pivot then Jaccard** (NLLB-200 or Sonnet) | **Cross-lingual sentence embedding similarity** (LaBSE) | **Topic-model alignment via anchored CorEx or BERTopic with multilingual embeddings** |
| Embedding | paraphrase-multilingual-MiniLM-L12-v2 (384d, mediocre cross-lingual) | **LaBSE** (109 langs, parallel-supervised) | **multilingual-E5-large** (SOTA on MIRACL/MTEB) | **Cohere multilingual-v3** (managed, strong) |
| Clustering | DBSCAN, fixed eps=0.35 | **HDBSCAN** (no eps, density-adaptive, stability scores) | **Agglomerative + silhouette tuning** (transparent dendrogram) | **BERTopic** (purpose-built for news topic modeling) |
| Frame detection | LLM free-form JSON, re-derived per story | **Boydstun Policy Frames Codebook** (15 frames, validated, supervised classifier) | **Media Frames Corpus + fine-tuned model** (Card et al.) | **Multi-LLM ensemble + Krippendorff α gating** |
| Validation | Citation lint + number reconciliation, single rater | **Inter-LLM agreement + α threshold per artifact** | **Held-out human gold set with published α** | **Crowdsourced annotators on Prolific (n≥3/item)** |
| Source set | 235 RSS feeds, no paywalled majors | **Mediacloud API** (full-text, deduped, since 2008) | **GDELT GKG join** (15-min latency, validated themes) | **Common Crawl News + paywall-aware extraction** |
| Longitudinal pin | Hash inputs only | **Open-weight frozen model** (Llama/Qwen pinned by checksum) | **Monthly canary-prompt drift detection** with logging | **Re-run historical corpus monthly; report drift delta** |

## 8. What already works (credit where due)

- The methodology pin for *inputs* is genuinely good practice and rarely seen in news-NLP projects. Hashing prompts and stopwords is the right instinct, even if the conclusion (longitudinal claims are now safe) is wrong.
- Pre-computing metrics and forcing the LLM to use them verbatim is the right architectural pattern. It blocks the most embarrassing class of hallucination (invented numbers).
- Citation-grounding lint (`validate_analysis.py`) blocks invented quotes — a real contribution.
- 235-feed multilingual ingest at $0/mo is impressive engineering. The pipeline (`ingest.py`, `extract_full_text.py`, `dedup.py`, `daily_health.py`) is clean.
- Body-text fallback to Wayback is thoughtful. 74% FULL is reasonable in absolute terms; the bias problem is in *which* 26% fail, not the rate.
- Tests (`tests.py:64 unit + tests_edge.py:18 + tests_e2e.py`) are solid for the engineering surface area.

These are the parts that survive contact with peer review. Build outward from them.

## 9. Constructive path forward

Pick one of two stances and commit to it. The current position is incoherent because it asserts (b) while only delivering (a):

**(a) Curated multilingual news reader with citation-grounded summaries.** Drop the "framing analysis" claim. Ship per-bucket summaries with quote grounding. No Jaccard, no exclusive vocab, no paradox/silence detection. Modest, honest, useful. The infrastructure already supports this — just remove the measurement layer.

**(b) Cross-cultural framing measurement instrument.** Then earn it. Concretely, before the next public claim:

1. Replace `[A-Za-z]{4,}` with a Unicode-aware tokenizer or a per-language tokenizer; bump to a major version. The current artifact set should be marked confounded and not used for comparative claims.
2. Replace cross-lingual Jaccard with translate-to-pivot or LaBSE-similarity. Either reduces the measurement to a defensible construct.
3. Adopt a closed frame codebook (Boydstun/Card). Re-derive per story is a prompt regression — remove it.
4. Add the five killer experiments (Section 6) as a recurring CI job. Publish the results next to every analysis.
5. Switch to (or also report) an open-weight model so the pin actually pins.
6. Disqualify stub-only buckets and paywalled-failure outlets from quantitative comparison; report them as text-only sources.

Either path is a real project. The current middle is a sophisticated façade over a metric that, on inspection, measures language identification and prompt style.
