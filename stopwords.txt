# English stopwords for the methodology pin.
#
# Pinned by meta_version.json -> tokenizer.stopwords_hash. Changes here
# require a meta_version bump: 'minor' if you only add words (back-compat),
# 'major' if you remove words (invalidates longitudinal vocab comparisons).
#
# Comments and blank lines are ignored. Words must be lowercase, len>=4
# (the tokenizer regex drops anything shorter), one per line OR
# whitespace-separated.
#
# Scope: as of meta_version 3.0.0 the metric layer reads pivot-language
# (English) fields produced by analytical/translate.py. This file is
# therefore an English-only list — that is correct, not a bug. It is
# expanded vs prior pins to cover NLTK / spaCy English plus news-attribution
# verbs ("said", "told", "reported", "according") and meta words ("today",
# "yesterday", "breaking") that swamp bucket-exclusive vocab metrics.
