# Measurement — the prior, held loosely

Adopt this as **vocabulary**; do not believe it yet. The whole taxonomy is
versioned (`framework_versions`) and expected to be confirmed, refined, or
replaced by the data.

## The validated spine — MFQ-2 (adopt as construct vocabulary)

`care`, `equality`, `proportionality`, `loyalty`, `authority`, `purity`.

- Independently verified as real and validated.
- **Do not drop Purity for convenience** — it's highly discriminating; cutting it
  must be a *data* decision, not a writing-difficulty one.

## Relational structure — Schwartz circumplex

Values arranged so adjacent = compatible, opposite = trades off. The skeleton the
trade-off network hangs on.

## Unvalidated extras (flag as such)

`honesty`, `autonomy` (and any others). Probe them, but **lightly** — they have
not earned axis status. Never silently fold them in as equals to the MFQ-2 six.

## Crossing dimensions — modifiers, not axes

- **Scope** — who's in the moral circle (`kin`/`stranger`/`group`/`humanity`).
- **Process** — rule-framed vs outcome-framed.

These are gate tags (`scope_variant`, `process_frame`), not axes.

## The novelty

You import the **constructs**, not the questionnaire items. The measurement is
through **revealed forced-choice**, not self-report. No one has validated these
foundations *as measured through forced-choice dilemmas* — that's the new
psychometrics. Each choice is one trade-off **edge**; the corpus over time is the
**network** (a defensible design bet, not field consensus).

## Held loosely — the discipline

- The corpus must be built so it **can surprise you** (`is_exploratory` items).
- Don't over-invest in the unvalidated extras.
- **Don't believe your own prior.** A confident reveal feels true whether earned
  or not; the framework is a hypothesis until the anchors and factor analysis say
  otherwise. In v1 the reveal is honestly hedged (see `apps/api/src/scoring.ts`).

See `analysis/README.md` for how the prior becomes a finding.
